# 外部ファイル変更の自動検出とリアルタイムUI反映

## 概要

Cork 外で Markdown ファイルが追加・変更・削除された際に、Cork の Kanban ボード UI に自動で反映されるようにする。
現在はユーザーがカードのステータスを変更したときのみタスク一覧が再読み込みされるが、この変更によりエディタ等での外部編集もリアルタイムに追跡できるようになる。

## 動機

- ユーザーは好みのエディタで Markdown ファイルを編集し、Cork で可視化したい
- 現在はファイルの変更を検知する仕組みがなく、外部編集が UI に反映されない
- ファイル追加・削除も同様に未検知であり、ユーザーが手動で操作する手段すらない
- リアルタイム反映により、エディタと Cork をシームレスに行き来できる

## ユースケース

1. ユーザーが VSCode 等でタスクの Markdown ファイルを編集 → Cork のカード内容が自動更新される
2. ユーザーがファイルを追加（新規タスク作成）→ Cork の該当カラムにカードが追加される
3. ユーザーがファイルを削除 → Cork からカードが消える
4. ユーザーが frontmatter の status を直接編集 → カードが別カラムに移動する

## 非目標（今回のスコープ外）

- ファイル新規作成のための Cork 内エディタ機能
- 競合解決（外部編集と Cork 内編集の同時発生時の挙動）
- ファイルリネームの追跡
- サブディレクトリの監視（フラットなディレクトリ構造を前提）
