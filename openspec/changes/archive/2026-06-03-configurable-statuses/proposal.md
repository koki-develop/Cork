# 設定で任意のステータスを定義できるようにする

## 概要

現在は `"todo"`, `"doing"`, `"done"` の 3 ステータスがハードコードされている。これを設定画面でユーザーが任意のステータスを定義できるように変更する。ステータスはラベルのみを持ち、ID のような概念は持たない。形式は `{ label: string }[]`。

## 動機

- 現在はステータスが固定されており、ユーザーのワークフローに合わせられない
- プロジェクトによって必要なステータスは異なる（例: "Backlog" / "In Progress" / "Review" / "Done"）
- 設定画面が既にあるので、そこでステータスを管理できるようにする

## ユースケース

1. ユーザーが設定画面を開き、ステータス一覧を確認・編集できる
2. ステータスを追加・削除・順序変更できる
3. 各ステータスに対応するカラムが動的に生成される
4. カードの "Move to" ボタンも動的に生成される
5. 設定変更後の既存タスクのステータスはそのまま維持される

## 非目標（今回のスコープ外）

- カラムの色設定
- ステータスごとの WIP 制限
- ステータス変更のバリデーション
- デフォルトステータスの指定
