# 設定画面の実装と作業ディレクトリの永続化

## 概要

Cork に設定画面を追加し、作業ディレクトリを永続化して次回起動時に自動的に復元できるようにする。
現在はアプリ起動のたびにディレクトリ選択が必要だが、この変更により一度設定すれば毎回選択する手間がなくなる。
また、設定画面からいつでも作業ディレクトリを切り替えられるようにする。

## 動機

- 現在は選択したディレクトリがメモリ上にしか保存されず、アプリを閉じると失われる
- アプリを開くたびにディレクトリ選択を強いられ、同じディレクトリを使い続けるユーザーにとっては不便
- ディレクトリを切り替える手段がない（アプリを閉じて再起動するしかない）
- 設定画面という UI の入り口を作ることで、今後他の設定項目を追加しやすくする

## ユースケース

1. ユーザーが初回起動時にディレクトリを選択 → 次回起動時は自動的に同じディレクトリが開かれる
2. ユーザーが設定画面から「ディレクトリを変更」ボタンをクリック → 新たにディレクトリを選択し、その場で Board が再初期化される
3. 保存されたディレクトリが存在しない（外部ストレージが取り外された等）場合 → ディレクトリ選択画面にフォールバックする

## 非目標（今回のスコープ外）

- 複数ディレクトリの管理
- テーマ設定など、ディレクトリ以外の設定項目
- 設定画面のリッチな UI（最低限の機能で十分）
- 初回起動時チュートリアルやオンボーディング
