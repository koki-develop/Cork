## ADDED Requirements

### Requirement: 設定パネルに Save / Cancel ボタンを表示しない

設定パネル (`SettingsPanel`) は確定 / 破棄を行うためのフッターボタンを保持しない。全ての操作はそれぞれ規定されたタイミングで即時永続化される。

#### Scenario: パネルを開いたときフッターボタンが存在しない

- **WHEN** ユーザーが設定パネルを開く
- **THEN** パネル下部に `Save` ボタンと `Cancel` ボタンのいずれも表示されない
- **AND** 「Unsaved changes」インジケータも表示されない

#### Scenario: Esc キーは閉じるのみ

- **WHEN** ユーザーが何かしらの操作をした後に Esc キーを押す
- **THEN** 設定パネルは閉じる
- **AND** 直前までに永続化された変更（フォルダ変更、ステータスの追加・編集・削除・並び替え）はそのまま保持される

#### Scenario: 背景クリックは閉じるのみ

- **WHEN** ユーザーがパネル外の暗転背景をクリックする
- **THEN** 設定パネルは閉じる
- **AND** 永続化済みの変更は破棄されない

### Requirement: Workspace directory はピック直後に即時反映する

ユーザーがディレクトリピッカーで新しいフォルダを選んだ瞬間に Tauri バックエンドへ反映し、フロントエンドの `useWorkspace.dir` も更新する。

#### Scenario: フォルダを選ぶと即時反映される

- **WHEN** ユーザーが Workspace Directory ボタンを押してダイアログから新しいフォルダを選ぶ
- **THEN** `set_workspace_directory(path)` が呼び出されてバックエンドの state と `settings.json` が更新される
- **AND** `useWorkspace.setDir(path)` が呼ばれて Board と設定パネルの表示パスが新しい値に切り替わる
- **AND** 設定パネルは閉じない

#### Scenario: フォルダ選択をキャンセルしたとき

- **WHEN** ユーザーがダイアログで「キャンセル」を選び `pick_directory` が `null` を返す
- **THEN** バックエンドにもフロントエンドにも変更は適用されない
- **AND** 設定パネルは開いたままで以前のディレクトリが表示される

#### Scenario: 同じフォルダを選び直したとき

- **WHEN** ユーザーが現在表示中と同じフォルダを選ぶ
- **THEN** `set_workspace_directory` の呼び出しはスキップしてもよいし呼んでもよいが、Board の表示パスは変化せず、ユーザーから見て副作用がない状態に保たれる

### Requirement: Status の削除はクリック直後に即時反映する

各ステータス行のゴミ箱ボタンを押した瞬間にローカル state から削除し、`save_statuses` を呼び出してストアに反映する。

#### Scenario: ゴミ箱ボタンで即時削除

- **WHEN** ユーザーがあるステータス行のゴミ箱ボタンを押す
- **THEN** 当該行が画面上から消える
- **AND** 残った行を `Vec<StatusEntry>` として `save_statuses` に渡し永続化される
- **AND** Board のカラム表示が更新後のステータスに追従する

#### Scenario: 全件削除

- **WHEN** ユーザーが表示されている全ステータスを順に削除する
- **THEN** 各クリックごとに `save_statuses` が呼ばれ、最終的に空配列が永続化される
- **AND** ユーザーは追加ボタンで新しいステータスを定義し直せる

### Requirement: Status の並び替えはドロップ直後に即時反映する

`@dnd-kit/react` の DnD でステータス行を移動した結果は、ドロップ完了時点で永続化される。ドラッグ中の中間状態は永続化しない。

#### Scenario: ドラッグして並び替えると即時反映

- **WHEN** ユーザーがステータス行をドラッグして別の位置にドロップする
- **THEN** `onDragEnd` が `canceled` でない場合に限り、新しい順序で `save_statuses` が呼び出される
- **AND** Board のカラム順も同じ順序で表示される

#### Scenario: ドラッグをキャンセル

- **WHEN** ユーザーがドラッグ中に Esc キーまたは元の位置でドロップして `onDragEnd.canceled` が `true` になる
- **THEN** ローカル state は開始時のスナップショットに巻き戻る
- **AND** `save_statuses` は呼ばれない

### Requirement: Status の追加・名称変更はフォーカスアウト時に即時反映する

`Add Status` ボタンや既存ラベルの入力はキー入力時点ではローカル state のみ更新し、`<input>` から `blur` した時点で永続化を試みる。

#### Scenario: 新規追加ステータスを入力してフォーカスを外す

- **WHEN** ユーザーが `Add Status` ボタンを押し、追加された空行に文字を入力した後フォーカスを外す
- **THEN** 入力したラベル（前後空白トリム後）を含む全ステータス配列で `save_statuses` が呼ばれる

#### Scenario: 既存ラベルを変更してフォーカスを外す

- **WHEN** ユーザーが既存のステータスラベルを書き換えてフォーカスを外す
- **THEN** トリム後のラベルで `save_statuses` が呼ばれて永続化される

#### Scenario: 入力中は永続化しない

- **WHEN** ユーザーが `<input>` に文字を入力中である
- **THEN** 各キー入力ごとに `save_statuses` は呼ばれない
- **AND** ローカル state のみが更新される

### Requirement: 空ラベル行はフォーカスアウト時に除外する

`Add Status` で追加された行や既存行のラベルを空にしてフォーカスを外したとき、ローカル state から該当行を取り除く（永続化対象にしない）。

#### Scenario: 空のまま追加行のフォーカスを外す

- **WHEN** ユーザーが `Add Status` を押した後、空のままその行からフォーカスを外す
- **THEN** その行はローカル state から除外される
- **AND** `save_statuses` は呼ばれない（既存ステータスに変更がないため）

#### Scenario: 既存ラベルを空にしてフォーカスを外す

- **WHEN** ユーザーが既存ラベルを全削除して空文字にしてフォーカスを外す
- **THEN** その行はローカル state から除外され、残ったステータスで `save_statuses` が呼ばれる

#### Scenario: 空白のみのラベル

- **WHEN** ユーザーが空白文字のみ（例: `   `）を入力してフォーカスを外す
- **THEN** トリム後に空になるためその行は除外される

### Requirement: 重複ラベルを検出したら永続化を中断する

ラベルの `onBlur` 時に、トリム済みかつ小文字化したラベルが他の行と一致する場合、永続化を行わずエラーメッセージを表示する。

#### Scenario: 重複したラベルでフォーカスを外す

- **WHEN** ユーザーが既存ラベル `Todo` と同じ値を別の行に入力してフォーカスを外す
- **THEN** `save_statuses` は呼ばれない
- **AND** パネル内に「重複ラベルは指定できない」旨のエラーメッセージが表示される
- **AND** ローカル state の `label` は入力された値のまま残り、ユーザーは修正を続けられる

#### Scenario: 重複を修正して再度フォーカスを外す

- **WHEN** 重複エラー表示中のユーザーがラベルを別の値に書き換えてフォーカスを外す
- **THEN** 重複が解消されていれば `save_statuses` が呼び出され、エラーメッセージは消える

#### Scenario: 大文字小文字違いの重複

- **WHEN** ユーザーが `todo` を入力して既存の `Todo` と被る
- **THEN** 重複として検出され永続化されない

### Requirement: 設定パネルは pending 状態を保持しない

旧仕様で存在した `pendingDir` 等の確定待ち state、`isDirty` フラグ、`Unsaved changes` インジケータを保持しない。

#### Scenario: 操作直後にパネルを閉じてもデータは保持される

- **WHEN** ユーザーがステータスを追加 / 削除 / 並び替えてから Esc または背景クリックでパネルを閉じる
- **THEN** 操作した内容はストアに永続化済みで、再度パネルを開いても同じ状態が見える
- **AND** 「Unsaved changes」のような表示は一度も現れない

#### Scenario: パネルを開き直したとき state がクリーン

- **WHEN** ユーザーが設定パネルを閉じてから再度開く
- **THEN** エラーメッセージや一時的な入力中の状態は持ち越されず、永続化済みのデータに基づいた初期表示になる
